# Global Styles

The cascade contract declares `@layer global` as its floor:

```
@layer global, base, variants, compounds, states, system, custom;
```

Every component style layer sits above it. Global styles — resets, body defaults, scrollbar rules, keyframe animations — belong there. `createGlobalStyles` puts them there using the same prop vocabulary as components: same shorthand props, same token refs, same extraction pipeline.

---

## createGlobalStyles

`createGlobalStyles` is returned alongside the system instance from `createSystem().build()`. It is not a standalone import — it is bound to your system's prop config, so it knows how to resolve the props you've registered.

```ts
export const { system: ds, createGlobalStyles } = createSystem()
  .addGroup('surface', { ...color, ...border, ...background })
  .addGroup('space', space)
  .build();
```

Call it once with a `GlobalStyleMap` to produce a `GlobalStyleBlock`. Export the result — the plugin discovers it by the `__brand` marker and emits it into `@layer global` at build time.

<TypeSignature
  name="createGlobalStyles"
  params={[{ name: 'styles', type: 'GlobalStyleMap' }]}
  returns="GlobalStyleBlock"
/>

The factory is called once at module evaluation time. The returned block is static — no runtime style injection, no re-evaluation per render.

### GlobalStyleMap

```ts
type GlobalStyleMap = Record<string, Record<string, any>>;
```

Top-level keys are CSS selectors. Values are style objects using system prop shorthand or raw CSS property names. Token references follow the same bare scale key form as component styles.

```ts
export const globalStyles = createGlobalStyles({
  '*, *::before, *::after': { boxSizing: 'border-box' },
  'html, body': { bg: 'bg', color: 'text' },
  a: {
    color: 'primary',
    textDecoration: 'none',
  },
  'a:hover': { textDecoration: 'underline' },
});
```

Prop shorthand (`bg`, `color`, `m`, `p`) resolves through the system's prop config — same scale lookups, same transform application, same token alias resolution as in `.styles()`. Raw CSS property names (`boxSizing`, `textDecoration`) pass through camelCase-to-kebab conversion only.

<Callout variant="info" title="Discovery by export">
  The plugin collects every export from your system module whose `__brand`
  equals `'GlobalStyleBlock'`. Multiple named exports are all collected and
  emitted in export order, joined inside one `@layer global` block. There is no
  registration call — exporting the block is sufficient.
</Callout>

## Reset & Global Layer Structure

At build time, the plugin runs a subprocess that loads your system module, reads every `GlobalStyleBlock` export, and calls `resolveGlobalStyles` from the extract pipeline. The pipeline resolves prop shorthand, scale lookups, transforms, and `{token.path}` aliases in values — then wraps the result in `@layer global`.

<CodeExample
  input={`// ds.ts
export const globalStyles = createGlobalStyles({
  '*, *::before, *::after': { boxSizing: 'border-box' },
  body: {
    m: 0,
    fontWeight: '400',
    lineHeight: '1.625',
  },
  'html, body': { bg: 'bg', color: 'text' },
  '::-webkit-scrollbar': { width: '3px', height: '3px' },
  '::-webkit-scrollbar-thumb': { bg: 'scrollbar' },
  '::selection': { bg: 'selection', color: 'selection.text' },
})`}
  output={`@layer global {
  *, *::before, *::after {
    box-sizing: border-box;
  }

body {
margin: 0;
font-weight: 400;
line-height: 1.625;
}

html, body {
background-color: var(--color-bg);
color: var(--color-text);
}

::-webkit-scrollbar {
width: 3px;
height: 3px;
}

::-webkit-scrollbar-thumb {
background-color: var(--color-scrollbar);
}

::selection {
background-color: var(--color-selection);
color: var(--color-selection-text);
}
}`}
/>

The emitted CSS contains no token values — only CSS variables. Token refs shift automatically across color modes because the variables are re-bound by `[data-color-mode]` selectors in the variable CSS layer.

### GlobalStyleBlock.serialize

<TypeSignature
  name="serialize"
  params={[
    { name: 'propConfig', type: 'Record<string, any>' },
    { name: 'transforms', type: 'Record<string, NamedTransform>' },
  ]}
  returns="GlobalStyleMap"
/>

`serialize` is called by the global styles resolution subprocess during `buildStart`. It receives the system's prop config (from `ds.toConfig().propConfig`) and the live transform functions (from `ds.toConfig().transforms`). The extract pipeline uses these to resolve prop shorthand and run named transforms against scale values before emitting CSS.

You do not call `serialize` directly. Export the `GlobalStyleBlock` and the plugin handles resolution.

## Keyframes & Animations

`@keyframes` rules use the at-rule string as the top-level key. The value is an object of keyframe stops, each of which is a style object accepting the same prop shorthand as any other rule.

<CodeExample
  input={`export const globalStyles = createGlobalStyles({
  // ... resets ...
  '@keyframes spotlight': {
    '0%, 100%': { textShadow: 'glow-text' },
    '50%': { textShadow: 'glow-text-strong' },
  },
})`}
  output={`@layer global {
  @keyframes spotlight {
    0%, 100% {
      text-shadow: 0 0 30px color-mix(in srgb, var(--color-glow) 40%, transparent), 0 0 80px color-mix(in srgb, var(--color-glow) 15%, transparent);
    }
    50% {
      text-shadow: 0 0 50px color-mix(in srgb, var(--color-glow) 70%, transparent), 0 0 120px color-mix(in srgb, var(--color-glow) 25%, transparent);
    }
  }
}`}
/>

Inside the structured form the frame bodies share the same vocabulary as the selector rules above them — prop shorthand (`textShadow: 'glow-text'`) resolves through the `shadows` scale because `createGlobalStyles` is bound to the system's prop config, and any `{colors.*}` references in the value resolve to CSS variables.

### Top-level `keyframes()` primitive

Alongside the structured `@keyframes <name>` form inside `createGlobalStyles`, declare keyframes via the top-level `keyframes()` factory. It returns a **branded collection** exposing one typed reference per named keyframe, intended for distributed authoring across files and packages.

```ts
import { keyframes } from '@animus-ui/system';

export const animations = keyframes({
  ember: {
    '0%, 100%': { textShadow: '{shadows.glow-text}' },
    '50%': { textShadow: '{shadows.glow-text-strong}' },
  },
  flow: {
    '0%': { backgroundPosition: '200% 0' },
    '100%': { backgroundPosition: '-200% 0' },
  },
  tallyPulse: {
    '0%, 100%': { transform: 'scale(1)' },
    '50%': { transform: 'scale(1.02)' },
  },
});
```

The return value is a record: `animations.ember`, `animations.flow`, and `animations.tallyPulse` are each branded `KeyframeRef` objects that carry their literal key at the type level. Typos fail at the collection access site with a clean "Property 'embr' does not exist" error, not at animation reference.

Each keyframe's CSS name is a deterministic FNV-1a content hash of its frame body (`animus-kf-<hash>`), stable across rebuilds. Identical frame bodies across keys dedupe into a single `@keyframes` block naturally.

<Callout variant="info" title="Frame body vocabulary narrows in `keyframes()`">
Inside a `keyframes()` frame body you can use CSS property names, raw values, and `{scale.key}` token references — the same syntax used everywhere else in Animus for theme-resolved values. Bare scale keys like `textShadow: 'glow-text'` (the form supported in `ds.styles()` and the structured `@keyframes <name>` selector inside `createGlobalStyles`) are NOT resolved here, because the factory is a top-level primitive and does not carry system prop-config context. Use the `{shadows.glow-text}` form instead.
</Callout>

The plugin discovers `keyframes()` collections via `__brand === 'Keyframes'` during system-module loading (same pattern as `createGlobalStyles` exports), iterates each named frame, and emits one `@keyframes <resolved-name> { ... }` block per key into `@layer global`.

### Referencing Keyframes in Components

Reference branded keyframe refs directly on `animationName`. The Rust extractor traces the binding back through imports to the originating `keyframes()` collection and substitutes the resolved name into the emitted CSS — your component's CSS stays fully static, with no runtime dynamic-style injection.

```tsx
import { animations, ds } from './ds';

export const EmberGlow = ds
  .styles({
    animationName: animations.ember,
    animationDuration: '4s',
    animationTimingFunction: 'ease-in-out',
    animationIterationCount: 'infinite',
    '@media (prefers-reduced-motion: reduce)': {
      animationName: 'none',
    },
  })
  .asElement('span');

export const Marquee = ds
  .styles({
    animationName: animations.flow,
    animationDuration: '5s',
    animationTimingFunction: 'linear',
    animationIterationCount: 'infinite',
  })
  .asElement('span');
```

The structured form inside `createGlobalStyles` (earlier example on this page) continues to work in parallel — you can author some keyframes through each path.

<EmberGlow>ember</EmberGlow> <TallyPulse>tally pulse</TallyPulse>

Because `@keyframes` rules land in `@layer global` and component styles land in layers above it, the keyframe declarations are always present when animation properties reference them.

<Callout variant="warn" title="Keyframe bindings must be statically resolvable">
  The extractor substitutes `animations.ember`-style member-expression
  references by tracing the `animations` binding back to its `keyframes(...)`
  factory call. Runtime-dynamic paths (shadowed bindings, values from a function
  call, dynamic computed access like `animations[key]`) fall through to the
  runtime `toString()` coercion fallback in dev, but fail the strict build. Keep
  the binding and key names literal at call sites.
</Callout>

## Type Reference

### GlobalStylesFactory

```ts
type GlobalStylesFactory = (styles: GlobalStyleMap) => GlobalStyleBlock;
```

Returned from `createSystem().build()`. Bound to the system instance's prop config and transforms. Each call produces an independent `GlobalStyleBlock`.

### GlobalStyleBlock

```ts
interface GlobalStyleBlock {
  __brand: 'GlobalStyleBlock';
  styles: GlobalStyleMap;
  serialize(
    propConfig: Record<string, any>,
    transforms: Record<string, NamedTransform>
  ): GlobalStyleMap;
}
```

Branded wrapper. The `__brand` field is the plugin's discovery signal — it scans all named exports of your system module and collects any value where `__brand === 'GlobalStyleBlock'`. The `.styles` property holds the original map. `.serialize()` is called during global styles resolution to run prop resolution and transform application before CSS emission.

### GlobalStyleMap

```ts
type GlobalStyleMap = Record<string, Record<string, any>>;
```

Keys are CSS selectors or `@keyframes` at-rule strings. Values are style objects accepting system prop names (resolved through prop config and scales) or raw camelCase CSS property names (converted to kebab-case verbatim).
