# System Props & Groups

System props are style props exposed directly on a component instance — the runtime surface that lets consumers pass token-bound values without writing inline styles or class names. Groups are named sets of related props registered on your design system instance. `.system()` opts a specific component into one or more groups.

<Callout variant="warn" title="Static values only">
All system prop values must be static string literals or static number literals. Dynamic values — template literals, computed variables, ternaries, function calls, spread objects — are skipped at extraction time. Responsive maps `{'{ _: 16, sm: 32 }'}` are valid when all keys and values are static.
</Callout>

---

## `.system()` — Enabling Groups on a Component

<TypeSignature
  name="system"
  generics="PickedKeys extends keyof GroupRegistry | Extract<keyof PropRegistry, string>"
  params={[{ name: 'config', type: 'Record<PickedKeys, true>' }]}
  returns="AnimusWithSystem"
/>

`.system()` is called after `.states()` (or after `.variant()` / `.compound()` if no states are defined). It accepts an object whose keys are group names registered on the design system instance and whose values are the literal `true`. The return type, `AnimusWithSystem`, exposes `.props()` and the terminal methods (`.asElement()`, `.asComponent()`, `.asClass()`).

Props from enabled groups are emitted into `@layer system` — above `@layer states`, below inline styles. This means system props can override base, variant, and state styles.

System props accept responsive values. Any group prop can receive a `MediaQueryMap` object: `{ _: 'column', sm: 'row' }`. The `_` key emits base styles; named keys emit `min-width` media queries matching your theme's breakpoints.

<CodeExample
  inputLabel="source"
  outputLabel="component props"
  input={`const Card = ds.styles({
  display: 'flex',
  flexDirection: 'column',
  bg: 'surface',
  p: 24,
  borderRadius: 'md',
})
  .system({ space: true, color: true })
  .asElement('div')

// Consumer usage
<Card
  bg="surface-raised"
  p={{ _: 16, sm: 24 }}
  color="text-muted"
/>`}
  output={`// @layer base — from .styles()
.animus-Card-a1b2 {
  display: flex;
  flex-direction: column;
  background-color: var(--colors-surface);
  padding: 1.5rem;
  border-radius: var(--radii-md);
}

// @layer system — from prop overrides
.animus-Card-sys-bg-a2b3 { background-color: var(--colors-surface-raised); }
.animus-Card-sys-color-c3d4 { color: var(--colors-text-muted); }
.animus-Card-sys-p-e5f6 { padding: 1rem; }
@media (min-width: 768px) {
  .animus-Card-sys-p-g7h8 { padding: 1.5rem; }
}`}
/>

You can also enable individual props by name rather than entire groups:

```tsx
const Box = ds.styles({})
  .system({ m: true, p: true, display: true })
  .asElement('div')
```

Any key in `GroupRegistry` (a group name) or in `PropRegistry` (a single prop name) is valid.

### `mode` — Per-Component Color Mode

Binds to the `mode` scale defined by `addModes()` on your theme. Setting `mode` on a component applies a `[data-color-mode]` attribute to that element's subtree, switching all contextual tokens to the named mode. No CSS property is written — the effect is entirely via the mode data attribute.

<ParamTable
  params={[

  .system({ mode: true })

<InvertedSection mode="light">...</InvertedSection>

## Combining Groups

Pass multiple group keys to a single `.system()` call. Only the groups listed become available as props — unused groups add no overhead.

<CodeExample
  inputLabel="source"
  outputLabel="available props"
  input={`const Box = ds.styles({})
  .system({
    space: true,
    color: true,
    layout: true,
    borders: true,
  })
  .asElement('div')

<Box
  p={24}
  mx="auto"
  bg="surface"
  color="text"
  maxW={960}
  border={1}
  borderColor="border"
  borderRadius="md"
/>`}
  output={`// Props available on <Box />:

//           maxHeight, maxH, verticalAlign, justifySelf, alignSelf,
