# compose()

`compose()` assembles independently-authored Animus components into a sealed, namespaced component family with shared variant propagation. Shared variant values flow through CSS descendant selectors — Root sets the value once, child slots inherit it automatically. No prop drilling, no runtime computation.

### context: true — how it works

When `context: true`, a `React.createContext` is created once at `compose()` call time. Root and all child wrappers are `forwardRef` components that either provide or consume it:

- **Root wrapper**: extracts the shared prop keys from its received props and injects a `Context.Provider` inside its rendered output, wrapping `props.children`. The Provider is placed _inside_ Root's rendered element so portaled React children remain in the React tree and receive the context.
- **Child wrappers**: read the context via `useContext` and spread inherited values under their own direct props — `{ ...inherited, ...ownProps }`. Direct props win.
- **CSS-only mode** (`context: false`): all wrappers are thin `forwardRef` shims with no hooks. No context is created.

```tsx
// context: true — Root provides, all children consume
const Dialog = compose(
  { Root: DialogRoot, Content: DialogContent, Overlay: DialogOverlay },
  { shared: { intent: true }, name: 'Dialog', context: true }
)

// DialogContent may be portaled outside DialogRoot's DOM subtree.
// CSS descendant selectors cannot reach it — context bridges the gap.
<Dialog.Root intent="danger">
  {ReactDOM.createPortal(
    <Dialog.Content>Destructive action</Dialog.Content>,
    document.body
  )}
</Dialog.Root>
```

### shared validated against Root variants

`shared` is type-checked against Root's `.variant()` config at compile time. Keys that do not exist on Root are type errors — not runtime errors.

```tsx
const CardRoot = ds
  .styles({ bg: 'surface' })
  .variant({ prop: 'density', variants: { sm: {}, md: {}, lg: {} } })
  .asElement('div');

compose(
  { Root: CardRoot, Body: CardBody },
  {
    shared: {
      density: true, // valid — Root has density
      // intent: true, // type error — Root has no intent variant
    },
  }
);
```

## Return Type: ComposedFamily

<TypeSignature
  name="ComposedFamily"
  generics="Slots extends Record<string, unknown>"
  params={[]}
  returns="{ [K in keyof Slots]: ForwardRefExoticComponent<SealedProps<Slots[K]>> }"
/>

`ComposedFamily<Slots>` maps each slot key to a `ForwardRefExoticComponent` with sealed props. The wrapper components forward refs to the original Animus component.

<Callout variant="info" title="Sealed output — no .extend()">
  Composed slot components have no `.extend()` method. `compose()` is a one-way
  door from builder-land to component-land. To extend a slot, extend the
  original slot component before passing it to `compose()`, or export the
  pre-composed component alongside the family for consumers to extend directly.
</Callout>

### SealedProps

`SealedProps<C>` is the per-slot prop type used inside `ComposedFamily`. It reads the `ConsumerProps` phantom brand from the original Animus component via direct indexed access — no conditional inference through `ForwardRefExoticComponent<infer P>`. This avoids depth explosion in environments with extra type-checking overhead (e.g. Next.js TS plugin).

`SealedProps` strips the `extend` method from the prop surface but preserves `ConsumerProps` and `VariantConfigBrand`, so `VariantPropsOf` continues to work on composed family members.

### Root slot wrapper behavior

- Accepts all props that the original Root component accepts, including shared variant keys.
- When `context: true`: extracts shared prop values and provides them via `Context.Provider` inside its rendered output.
- `displayName` is set to `"FamilyName.Root"`.

### Child slot wrapper behavior

- Accepts all props that the original child component accepts, including shared variant keys (for per-slot override).
- When `context: true`: reads family context, merges inherited values under direct props — `{ ...inherited, ...ownProps }`.
- When `context: false`: thin `forwardRef` wrapper with no hooks. CSS-only propagation.
- `displayName` is set to `"FamilyName.SlotName"`.

### Overriding shared values on a child

A prop passed directly to a child slot overrides the inherited value. The extractor emits a second CSS rule per shared value — a direct `[data-*]` selector on the child class — which takes precedence in the cascade over the descendant selector from Root.

```tsx
// density="sm" on Root propagates via CSS to Header and Body
// density="lg" on Body overrides the inherited value for Body only
<Card.Root density="sm">
  <Card.Header>Compact header</Card.Header>
  <Card.Body density="lg">Spacious body</Card.Body>
</Card.Root>
```

### VariantPropsOf on composed slots

`VariantPropsOf<C>` works on any slot from a `ComposedFamily`. The `VariantConfigBrand` is preserved through `SealedProps` — `ComposedFamily` strips only the `extend` method, not the brands.

```tsx
import type { VariantPropsOf } from '@animus-ui/system';

type CardDensity = VariantPropsOf<typeof Card.Root>;
// → { density?: 'sm' | 'md' | 'lg' }

function CardSection({
  density,
  children,
}: VariantPropsOf<typeof Card.Root> & { children: React.ReactNode }) {
  return (
    <Card.Root density={density}>
      <Card.Header>Section</Card.Header>
      <Card.Body>{children}</Card.Body>
    </Card.Root>
  );
}
```

For the full consumer prop surface of a slot — including HTML element props and state props — use `ComponentPropsWithRef<typeof Card.Root>`.
