# Builder Chain

The builder chain is a type-state machine. Each method returns a narrower class that removes the methods preceding it in the cascade. Ordering is enforced at the type level — calling `.variant()` after `.compound()` is a compile error.

---

## Type-State Machine

The primary chain consists of seven classes arranged in a backwards-inheritance hierarchy. Each child class extends the class that represents the next stage, inheriting only the methods that remain callable. The chain maps directly to CSS `@layer` order.

<ChainStep
  steps={[
    {
      label: 'ds.styles()',
      layer: 'base',
      description:
        'Base styles applied unconditionally to every instance. Entry point for the chain.',
      code: `const Button = ds
  .styles({ display: 'inline-flex', px: 16, py: 8 })`,
      repeatable: false,
    },
    {
      label: '.variant()',
      layer: 'variants',
      description:
        'Adds a named variant axis. Variant prop values are static string keys. Can be called multiple times to add additional axes.',
      code: `  .variant({
    prop: 'size',
    defaultVariant: 'md',
    variants: {
      sm: { px: 8, py: 4, fontSize: 13 },
      md: { px: 16, py: 8, fontSize: 15 },
      lg: { px: 24, py: 12, fontSize: 17 },
    },
  })`,
      repeatable: true,
    },
    {
      label: '.compound()',
      layer: 'compounds',
      description:
        'Styles applied when multiple variant conditions are satisfied simultaneously. Requires at least one variant to be defined.',
      code: `  .compound(
    { size: 'lg', intent: 'danger' },
    { fontWeight: 700, letterSpacing: '0.02em' }
  )`,
      repeatable: true,
    },
    {
      label: '.states()',
      layer: 'states',
      description:
        'Boolean prop-driven style blocks. Each key becomes a boolean prop on the component.',
      code: `  .states({
    disabled: { opacity: '0.4', pointerEvents: 'none' },
    loading: { cursor: 'wait' },
  })`,
      repeatable: false,
    },
    {
      label: '.system()',
      layer: 'system',
      description:
        'Opts in individual props or named groups from the design system prop registry as passthrough props on the component.',
      code: `  .system({ color: true, typography: true })`,
      repeatable: false,
    },
    {
      label: '.props()',
      layer: 'custom',
      description:
        'Defines custom prop configurations outside the system registry. Each entry is a full prop spec with property, scale, and transform.',
      code: `  .props({
    gap: { property: 'gap', scale: 'space' },
  })`,
      repeatable: false,
    },
  ]}
/>

<Callout variant="info" title="Backwards inheritance">
  The class hierarchy is inverted from a normal OOP hierarchy. `AnimusWithAll`
  is the final stage and the most restricted — it has terminals but no chain
  methods. `Animus` is the entry point with the most methods available. Each
  stage removes the methods that came before it.
</Callout>

### Class hierarchy

| Class                 | Adds                                                        | Inherits terminals from |
| --------------------- | ----------------------------------------------------------- | ----------------------- |
| `Animus`              | `.styles()`                                                 | —                       |
| `AnimusWithBase`      | `.variant()`                                                | —                       |
| `AnimusWithVariants`  | `.variant()` (repeatable), `.compound()`                    | —                       |
| `AnimusWithCompounds` | `.compound()` (repeatable), `.states()`                     | —                       |
| `AnimusWithStates`    | `.system()`                                                 | —                       |
| `AnimusWithSystem`    | `.props()`                                                  | —                       |
| `AnimusWithAll`       | `.asElement()`, `.asComponent()`, `.asClass()`, `.extend()` | terminal                |

## Entry: `Animus` class

`Animus` is created by `createSystem().build()` and exposed as the `ds` object. It holds the prop registry and group registry for the entire design system.

<TypeSignature
  name="Animus"
  generics="PropRegistry, GroupRegistry"
  params={[]}
  returns="Animus<PropRegistry, GroupRegistry>"
/>

The `ds` instance is a singleton per system. Do not instantiate `Animus` directly — use `createSystem().build()`.

<Callout variant="tip" title="Repeatable">
  `.variant()` can be called multiple times. Each call adds a new variant axis.
  Two calls with the same `prop` value merge — later call's variants extend the
  earlier ones. This enables compositional variant building across inheritance
  chains.
</Callout>

<Callout variant="warn" title="Prop values are static strings">
  Variant prop values are always static string keys. Responsive arrays (`['sm',
  'lg']`) are not valid for variant props. Use `.states()` or system props for
  responsive behavior.
</Callout>

### `.asClass()`

<MethodCard
  name="asClass"
  description="Returns a class resolver function instead of a React component. No React dependency."
  returnType="(props?: Record<string, unknown>) => string"
  available="styles (any terminal stage)"
  example={
    <TypeSignature
      name="asClass"
      params={[]}
      returns="(props?: Record<string, unknown>) => string"
    />
  }
/>

`.asClass()` produces a plain function that accepts an optional props object and returns a class name string. Useful for non-React contexts, class composition utilities, or server-side rendering without JSX.

```ts
const buttonClass = ds
  .styles({ display: 'inline-flex', px: 16 })
  .variant({
    variants: { primary: { bg: 'primary' }, ghost: { bg: 'transparent' } },
  })
  .asClass();

// Usage:
const cls = buttonClass({ variant: 'primary' });
// → 'animus-Button-abc123 animus-Button-variant-primary-def456'
```

<Callout variant="warn" title="No .extend() on asClass output">
  The function returned by `.asClass()` does not have an `.extend()` method. To
  create a variant of a class-based definition, use `.extend()` before calling
  `.asClass()`.
</Callout>

## Extension chain

`.extend()` is available on `AnimusWithAll` directly, and is attached to the result of `.asElement()` and `.asComponent()`. It returns an `AnimusExtended` instance, which begins the extension chain.

### `AnimusExtended`

The extension chain lifts the ordering constraint. `AnimusExtended` exposes `.styles()`, `.variant()`, `.states()`, `.compound()`, `.system()`, and `.props()` in any order. Each call merges into the accumulated configuration rather than replacing it.

```ts
const PrimaryButton = ds
  .styles({ display: 'inline-flex', px: 16, py: 8 })
  .variant({
    variants: { solid: { bg: 'primary' }, ghost: { bg: 'transparent' } },
  })
  .asElement('button');

// Extension — no ordering constraint:
const LargeButton = PrimaryButton.extend()
  .styles({ px: 24, py: 12 }) // merges with base styles
  .variant({
    // adds or merges variant axes
    prop: 'size',
    variants: { lg: { fontSize: 17 } },
  })
  .states({ loading: { opacity: '0.6' } }) // can come after variant
  .asElement('button');
```

### `AnimusExtendedWithAll`

The terminal stage of the extension chain. Has the same terminal methods as `AnimusWithAll`: `.asElement()`, `.asComponent()`, `.asClass()`, and `.extend()`.

| Class                        | Methods available                                                                         |
| ---------------------------- | ----------------------------------------------------------------------------------------- |
| `AnimusExtended`             | `.styles()`, `.variant()`, `.states()`, `.compound()`, `.system()`, `.props()`, terminals |
| `AnimusExtendedWithBase`     | `.variant()`, `.states()`, `.compound()`, `.system()`, `.props()`, terminals              |
| `AnimusExtendedWithVariants` | `.variant()` (repeatable), `.states()`, `.compound()`, `.system()`, `.props()`, terminals |
| `AnimusExtendedWithStates`   | `.system()`, `.props()`, terminals                                                        |
| `AnimusExtendedWithSystem`   | `.props()`, terminals                                                                     |
| `AnimusExtendedWithAll`      | terminals only                                                                            |

<Callout variant="tip" title="No ordering constraint in the extension chain">
  In `AnimusExtended`, `.variant()` and `.states()` can be called in either
  order. `.styles()` merges (deep merge) rather than replacing the inherited
  base styles. This is the intended way to override a specific slice of a parent
  component's styles.
</Callout>

<Callout variant="warn" title="Sealed compose() output">
  Components created by `compose()` do not have an `.extend()` method. The
  extension chain is only available on components created by `.asElement()` and
  `.asComponent()`.
</Callout>

## `.extend()` method

<MethodCard
  name="extend"
  description="Enters the extension chain. Available on AnimusWithAll and on asElement/asComponent output."
  returnType="AnimusExtended"
  example={
    <TypeSignature
      name="extend"
      params={[]}
      returns="AnimusExtended<PropRegistry, GroupRegistry, BaseStyles, Variants, States, ActiveGroups, CustomProps>"
    />
  }
/>

`.extend()` is a zero-argument method. It creates a new `AnimusExtended` instance pre-loaded with all accumulated configuration from the parent. Calling it on an `asElement` or `asComponent` result is equivalent to calling it on the `AnimusWithAll` instance before the terminal.
