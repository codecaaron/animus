## Why

When extraction bails on a chain (non-static expression, unknown method) or skips individual properties (variable reference, function call), the diagnostics are generated by the Rust crate but never reach the consumer. The project analyzer discards skip warnings (`_skip_warnings`). The Vite plugin never reads bail reasons from the manifest. A new user whose component silently loses styles gets zero console output unless they enable `ANIMUS_DEBUG=1` — which they don't know exists.

Elimination warnings (reconciliation pruning) ARE always-on. But extraction bail and per-property skip warnings are invisible. These are the most common "why is my style missing?" diagnostics.

## What Changes

- **Surface bail and skip warnings by default**: The project analyzer SHALL include per-component bail reasons and per-property skip warnings in the manifest. The Vite plugin SHALL print these as warnings (not gated by verbose mode), matching the existing elimination warning pattern.
- **Structured diagnostics in manifest**: The manifest gains a `diagnostics` array alongside the existing `report` object. Each diagnostic includes the component binding name, the file path, the kind (bail/skip), and the message.
- **No new CLI flags**: These warnings use the existing `warn()` helper (goes through Vite logger). Suppressible via Vite's `logLevel` config if consumers find them noisy.

## Capabilities

### New Capabilities
- `extraction-diagnostics`: Bail and skip warnings surfaced from Rust crate through manifest to Vite plugin console output.

### Modified Capabilities
- `rust-extraction-pipeline`: Project analyzer includes skip warnings and bail reasons in manifest output.
- `vite-extraction-plugin`: Plugin reads and prints extraction diagnostics from manifest (always-on, not verbose-gated).

## Impact

- **`packages/extract/src/project_analyzer.rs`**: Stop discarding `_skip_warnings`. Include bail reasons and skip warnings in manifest JSON.
- **`packages/vite-plugin/src/index.ts`**: Read `diagnostics` from manifest, print via `warn()` (same pattern as elimination warnings).
- **No runtime changes**: Diagnostics are build-time only.
- **Console output**: Users will see warnings like `[animus] ⚠ Button: skipped property 'color' (variable reference)` and `[animus] ⚠ DynamicBox: bail (function call in style object)`.
